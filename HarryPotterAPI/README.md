## Проверка Room
В Room хранится таблица любимых персонажей (favourite_character), в которую добавляется персонаж при нажатии сердечка в деталях персонажа

Как проверить: открыть приложение, нажать на одного/нескольких персонажей и в деталях у них нажать на сердечко. Закрыть приложение, открыть заново, нажать на "Любимые". Добавленные персонажи останутся даже после перезапуска.

### 7 unit-тестов
* Корректное начальное состояние экрана
* Корректное обновление запроса при наборе в строку поиска
* Успешная загрузка данных
* Ошибка загрузки данных
* Корректная фильтрация по дому
* Корректная обработка пустого результата
* проверка на retry() после ошибки

### Интеграционные тесты
* Выбирается нужный персонаж при клике на его карточку

## Финальная версия
* Добавлена сущность пользователя, который хранится в DataStore в JSON, в ней лежит id персонажа, оценки за заклинания, выбранная тема и локация
* Добавлены экраны создания персонажа - заполнение полей и квиз на выбор факультета
* Добавлены экраны изучения заклинаний - список и мини-игра
* Добавлен экран настроек
* Добавлены темы и задний фон - темы меняются в зависимости от выбранного факультета или в настройках, задний фон зависит от локации
* Добавлены короткие видео на создание персонажа, распределяющую шляпу и отчисление (сброс пользователя)
* Персонажи и заклинания получаются из api и хранятся в двух таблицах в бд, при заходе в приложение грузятся оттуда, настройки хранятся локально в JSON через DataStore
